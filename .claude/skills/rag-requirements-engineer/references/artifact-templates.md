# Шаблоны артефактов — Формальные спецификации

Готовые шаблоны для генерации артефактов. Каждый шаблон содержит все обязательные поля с описанием заполнения. Использовать как основу — заменять `[плейсхолдеры]` конкретными значениями, полученными в диалоге.

---

## Шаблон 1: БТ-карточка (Бизнес-требования)

```markdown
# БТ-[номер]: [Название бизнес-требования]

**Версия:** 1.0
**Дата:** [дата создания]
**Автор:** [роль автора]
**Статус:** Черновик | Согласовано | В работе | Закрыто

---

## Бизнес-цель

[Формулировка бизнес-ценности. Начинать с глагола: "Обеспечить...", "Снизить...", "Увеличить..."
НЕ технические решения — только бизнес-результат.

Пример: "Сократить время поиска ответа в технической документации с 15 минут
до 30 секунд для инженеров службы поддержки."]
```

## Матрица трассируемости

Использовать для проверки полноты покрытия после генерации всех артефактов.

```markdown

| БТ | ФТ | ТЗ-компонент | TC |
|----|----|--------------|----|
| БТ-001 | ФТ-001 | ТЗ: Ingestion | TC-001 |
| БТ-001 | ФТ-002 | ТЗ: Chunking | TC-002 |
| БТ-001 | ФТ-003 | ТЗ: Embedding | TC-003 |
| БТ-001 | ФТ-004 | ТЗ: Indexing | TC-004 |
| БТ-001 | ФТ-005 | ТЗ: Retrieval | TC-005, TC-006 |
| БТ-001 | ФТ-006 | ТЗ: Reranking | TC-007 |
| БТ-001 | ФТ-007 | ТЗ: Generation | TC-008, TC-009 |
| БТ-001 | ФТ-008 | ТЗ: Evaluation | TC-010 |

Граничные случаи:
| Сценарий | TC | Метрика |
|----------|----|----|

| Мультиязычный запрос | TC-013 | recall@10 |
| Нагрузочный тест | TC-014 | latency p95 |
```

## Правила именования

| Артефакт | Формат | Пример |
|----------|--------|--------|
| Бизнес-требование | БТ-[трёхзначный номер] | БТ-001 |
| Функциональное требование | ФТ-[трёхзначный номер] | ФТ-007 |
| Тест-кейс | TC-[трёхзначный номер] | TC-013 |
| Версия документа | [major].[minor] | 1.2 |

Нумерация сквозная и неизменная. Удалённые артефакты не переиспользуют номер — помечать статусом "Отменено".
