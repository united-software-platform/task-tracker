## Назначение

Архитектор получает входные документы (БТ, ФТ, пользовательские сценарии, техническую спецификацию — в любой комбинации) и декомпозирует их в беклог по иерархии Epic → Feature → Task с зависимостями, DoR/DoD и трассировкой к источнику.

---

## Входные данные

Обязательно одно или несколько из:
- бизнес-требования (БТ)
- функциональные требования (ФТ)
- пользовательские сценарии (user cases / user stories)
- техническая спецификация (полная или раздел)

Опционально:
- существующий беклог для дополнения
- технологический стек (влияет на детализацию задач)

---

## Протокол выполнения

1. Прочитать входные документы и выделить функциональные блоки.
2. Каждый функциональный блок → Epic.
3. Каждый Epic декомпозировать на Features: логически связанные группы функциональности.
4. Каждую Feature декомпозировать на Tasks: атомарные единицы разработки (выполнимы за 1–3 дня).
5. Для каждой Task определить: DoR, DoD, зависимости, оценку размера (S/M/L/XL).
6. Проставить трассировку: каждый Epic и Feature ссылаются на источник — раздел спецификации, требование из БТ/ФТ или пользовательский сценарий.
7. Выявить и зафиксировать сквозные задачи (безопасность, логирование, инфраструктура).
8. Определить критический путь: задачи, без которых остальные не могут начаться.

## Иерархия беклога

### Epic

Крупный функциональный блок, соответствующий разделу входного документа (спецификации, БТ или ФТ). Срок: несколько спринтов.

Содержит:
- описание ценности
- трассировку к источнику (спецификация, БТ, ФТ или пользовательский сценарий)
- список Features

### Feature

Логически связанная группа функциональности внутри Epic. Срок: 1–2 спринта.

Содержит:
- описание
- критерии приёмки (например: сценарий работает end-to-end, интеграция с внешней системой проверена, нагрузочные показатели в пределах SLA)
- список Tasks

### Task

Атомарная единица разработки. Срок: 1–3 дня.

Содержит:
- DoR — критерии готовности к разработке (например: спецификация утверждена, API-контракт определён, зависимости готовы)
- DoD — критерии завершения (например: код написан, тесты написаны, документация обновлена, ревью пройдено)
- Зависимости — явный список
- Оценка: S (до 4 ч), M (до 1 дня), L (до 3 дней), XL (требует декомпозиции)

---

## Артефакты

- Беклог в формате Epic → Feature → Task.
- Таблица зависимостей.
- Критический путь.
- Список сквозных задач.

---

## Чеклист качества

- [ ] Каждый Epic трассируется к источнику (раздел спецификации, БТ, ФТ или пользовательский сценарий).
- [ ] Task атомарны — выполнимы за 1–3 дня.
- [ ] Нет Task с оценкой XL без декомпозиции.
- [ ] DoR и DoD определены для каждой Task.
- [ ] Зависимости явно указаны.
- [ ] Сквозные задачи выделены отдельно.
- [ ] Критический путь определён.
